# 스펙 부록 — 다건(배치) 비교 v1

> 정본 스펙 [spec-hex-comparator.md](spec-hex-comparator.md)의 **단건** 구조를 확장한다.
> 비교/판정 의미(§5·§6)와 불변식(byte-for-byte, 보정 없음, 클라이언트 전용, 외부요청 0)은 **그대로 유지**.
> 이 문서는 마스터 세션(=사용자) 결정사항을 명문화한 것.

## 1. 목적

여러 개의 As-Is/To-Be 테이블 쌍을 **한 번에** 비교하고, 결과를 **(요약) 대시보드 + (세부) row 증거** 2단으로 본다.

## 2. 입력 / 페어링

- 업로드존 2개: **As-Is 묶음 / To-Be 묶음**.
- 각 존은 **① 폴더 통째 선택(webkitdirectory) ② 개별 파일 멀티선택 ③ 드래그&드롭** 지원. `.csv`만 수집.
- **페어링 = 파일명 정확히 일치**로 짝지음 (실데이터는 파일명 동일·디렉토리만 다름). 예: As-Is `고객.csv` ↔ To-Be `고객.csv`.
- 한쪽에만 있는 파일명 → **미매칭(unmatched)** 으로 수집·경고.
- 같은 그룹 내 파일명 중복 시 첫 항목 사용.

## 3. 키 / 설정

- **PK 선택 UI 없음 — 키는 쌍마다 자동 도출(autoKey).**
  - 이름 힌트(`unique_number`/`id`/…) 단일 컬럼이 유일하면 그걸 키로.
  - 아니면 **앞 컬럼부터 유일해질 때까지 확장 → 복합키 자동 지원** (예: `지점코드 + 계좌번호`).
  - 단일·복합 모두 지원. 키 컬럼은 비교(검사) 대상에서 자동 제외.
- **전역 기본**: 바이트 diff 표시. 제외 컬럼 기본 없음.
- **테이블별 override(드릴인 고급)**: 키 컬럼 추가/제거 · 제외 컬럼 조정 → 그 쌍만 재계산, 전체 판정 갱신.

## 4. 판정

- 쌍마다 단건 규칙(§6) 그대로: `PASS ⟺ 불일치0 + 누락0 + 초과0`.
- **전체 판정: 모든 쌍이 PASS AND 미매칭 파일 0 → 전체 PASS. 그 외 FAIL.**
  - 즉 **짝 없는 테이블(한쪽에만 존재)은 전체 FAIL** 사유 (테이블 통째 누락/초과 = 이행 문제).

## 5. 결과 화면 (2단)

**[요약] 대시보드**
- 상단: 전체 PASS/FAIL + 집계(테이블 수 · 통과/실패/미매칭 · 전체 일치/불일치/누락/초과 row 합계).
- 테이블별 목록: 이름 · 판정 · 일치/불일치/누락/초과 건수 · PK중복 경고. **PASS 테이블도 모두 노출**(실패만 X).
- 미매칭 파일 목록 표시.
- 행 클릭 → 세부.

**[세부] 드릴인 (기존 단건 화면 재사용)**
- 선택 테이블의 판정·집계·상세 row 목록.
- **일치 row도 동등하게 표시** (스펙 §8-D의 "기본 문제만"을 **"기본 전체"로 갱신**). 데이터 소량(≤100건)이라 전부 렌더.
- row 펼치면 컬럼별 As-Is vs To-Be 값 — **불일치는 다른 바이트 하이라이트, 일치는 동일 값 확인**.
- 테이블별 설정 override + "목록으로".

## 6. 내보내기

- **배치 요약 CSV**: 테이블당 1행 — `table, verdict, total, matched, mismatched, missing_in_tobe, extra_in_tobe, pk, excluded_columns`. 상단 메타에 전체 판정/집계/미매칭.
- 테이블 상세 CSV: 드릴인에서 기존 단건 내보내기 그대로.

## 7. 비기능 (유지)

- 단일 오프라인 HTML · 클라이언트 전용 · 외부요청 0 · 엔진(core.js) 재사용.
- 데이터 소량 전제(테이블당 ≤100건) → 동기 일괄 처리(진행률 불필요).
- ko/ja/en 다국어 유지.
