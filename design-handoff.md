# Claude Design 핸드오프 — As-Is/To-Be 데이터 비교 검증 도구

> 이 문서는 **Claude Design에게 그대로 건네는 프롬프트**다.
> 기능 스펙은 [spec-hex-comparator.md](spec-hex-comparator.md) §8(화면 요구사항), §10(분업 경계)이 정본.
> 로직은 이미 구현·검증 완료(테스트 47/47 통과, 외부 요청 0 확인). **너는 비주얼만 입힌다.**

---

## 0. 한 줄 요청

엔터프라이즈 마이그레이션 **검증 결과서**를 위한 단일 화면 웹 도구의 비주얼 디자인을 만들어줘. 색·타이포·여백·레이아웃·모션·아이콘은 전부 네 몫이다. 단, 아래 **절대 제약**과 **통합 방식**을 지켜야 산출물이 그대로 붙는다.

## 1. 제품 맥락 (톤을 결정하는 정보)

- **무엇**: 메인프레임(As-Is) → 오픈시스템(To-Be) 데이터 마이그레이션이 제대로 됐는지 CSV 2개를 byte-for-byte 비교해 **PASS / FAIL**을 책정하는 도구.
- **누가**: 마이그레이션 검증 담당자(개발/QA/감리). 수백~수천 row의 헥스 문자열을 눈으로 대조해야 하는 사람.
- **왜 존재하나**: 기존 도구는 평문 로그라 "어느 row의 어느 값이 다른지"와 "최종 합격 여부"가 한눈에 안 들어온다. **이 도구의 존재 이유 = 판정과 차이를 즉시 인지시키는 것.**
- **무드**: 신뢰·정밀·감사(audit)에 적합한 차분하고 단정한 톤. 장식보다 가독성과 명료함. 결과 화면은 단호해야 한다(합격/불합격은 타협 없는 이분법).

## 2. 절대 제약 (어기면 산출물 폐기)

1. **헥스/원시 데이터 값은 반드시 등폭(monospace)** — `.mono`. 자리 대조 가독성의 핵심. 이거 하나만은 스펙이 못박은 제약이다.
2. **PASS/FAIL 판정이 화면에서 가장 먼저·가장 강하게 인지**돼야 한다(§8-B). 결과 화면의 단일 주인공.
3. **외부 요청 0을 유지**한다. 산출물은 폐쇄망용 단일 오프라인 `index.html`이다. → **웹폰트 CDN·원격 아이콘·외부 이미지·구글폰트 링크 절대 금지.** 폰트는 시스템 폰트 스택, 아이콘은 인라인 SVG/유니코드/CSS로만. 모든 자산은 인라인.
4. **로직·판정 의미·기능을 바꾸지 않는다.** 판정 규칙(PASS⟺불일치0+누락0+초과0), 비교 방식, 화면에 표시되는 정보 항목을 더하거나 빼지 않는다. 시각 표현만 바꾼다.
5. **색에만 의존한 정보 전달 금지**(접근성). PASS/FAIL과 diff 하이라이트는 색+텍스트/형태/아이콘으로 이중 인코딩. 검증 도구라 red/green 색약 사용자도 판정을 오인하면 안 된다.

## 3. 너의 자유

색 팔레트, 타이포 스케일, 여백/그리드, 카드·보더·그림자, 레이아웃(단일 스크롤/탭/스텝퍼 등), 마이크로 인터랙션·트랜지션, 아이콘 스타일, 다크모드 여부, 빈/로딩/에러 상태의 시각 처리 — 전부 자유. 2번 제약을 깨지 않는 선에서.

## 4. 통합 방식 (이대로 줘야 그대로 붙는다)

현재 구조: `src/core.js`(로직, 손대지 말 것) + `src/app.js`(UI, className 훅 보유) + `src/styles.css`(중립 placeholder) → `build.mjs`가 단일 `index.html`로 인라인.

**1순위 산출물: 새 `src/styles.css` (CSS-only 교체).**
아래 className 계약에만 스타일을 입히면 마크업/로직 변경 없이 바로 빌드된다. 가장 안전하고 빠른 경로.

**마크업 변경이 꼭 필요하면**: 임의로 고치지 말고 "필요한 className 추가/구조 변경 목록"으로 요청해줘 → Claude Code가 `src/app.js`에 반영한다. (예: "verdict 안에 아이콘용 `<span class="verdict__icon">` 슬롯 추가 요청")

**className 계약 (현재 존재하는 훅 — 이 이름들에 스타일을 건다):**

```
레이아웃   .app .app__header .app__headtext .app__title .app__subtitle .card .card__title .muted .small .mono
언어       .langselect  (헤더 우측 언어 선택 select. 설정·결과 양쪽에 상주)
업로드     .uploads .drop .drop--over .drop--loaded .drop__role .drop__name .drop__hint .colchips .chip
설정       .field .field__label .toggles .toggle .toggle--on .toggle--off .toggle--pk .checkrow
액션/메시지 .btn .btn--secondary .btnrow .error .warn .render-note
판정(B)    .verdict .verdict--pass .verdict--fail .verdict__badge .verdict__reason .verdict__meta
집계(C)    .metrics .metric .metric__num .metric__label
상세(D)    .filters .detail-row .detail-row__head .detail-row__head--plain
           .badge .badge--match .badge--mismatch .badge--only
           .detail-row__key .detail-row__preview .detail-row__body
diff(D)    .diffpair .diffpair__col .diffpair__sides .side .side__label .side__val
           .u  .u.diff  .u.diff.residual  .empty
```

상태 modifier 의미:
- `--over`(드래그 중) `--loaded`(파일 인식됨)
- `.toggle--on`(검사 중) `.toggle--off`(제외됨) `.toggle--pk`(PK, 비활성)
- `.badge--match/--mismatch/--only`(일치/불일치/한쪽만)
- `.u.diff`(다른 유닛) `.u.diff.residual`(길이 차로 한쪽에만 남은 잔여 유닛) `.empty`(빈 값 마커 ∅)

## 5. 화면별 정보·우선순위 (스펙 §8 요약 — 항목을 빼지 말 것)

**공통 헤더** (`.app__header`, 설정·결과 양쪽 상주)
- 좌측: 제목/부제(`.app__headtext`). 우측: **언어 선택**(`.langselect` — 한국어/日本語/English).
- 기본 언어는 브라우저 설정 자동감지(ko/ja → 해당, 그 외 en). 선택은 표시 전용이며 비교·판정·CSV 결과를 바꾸지 않는다.

**화면 A — 업로드 & 설정** (작업 시작점, 차분하게)
- As-Is/To-Be 두 파일 업로드존(드래그앤드롭+클릭). **어느 쪽이 As-Is/To-Be인지 명확히 구분**.
- 업로드 후 각 파일의 인식된 컬럼 목록(칩) 확인 가능.
- PK 선택(드롭다운, 자동 추측값 표시).
- 검사/제외 컬럼 토글 — 기본 전체 검사. **각 컬럼의 검사중/제외됨 상태가 한눈에 구분**. PK는 별도 표시.
- 바이트 단위 diff 표시 옵션(체크박스).
- "비교 실행 및 판정" 주 액션.
- 입력 오류(미업로드/PK미선택/검사컬럼0개/공통컬럼0개) 명확한 메시지.

**화면 B — 판정** (결과의 주인공, 최상단·최강조)
- PASS/FAIL 배지가 압도적으로 먼저 읽혀야 함.
- 근거 한 줄: PASS="완전 일치"(제외 컬럼 있으면 명시) / FAIL=사유별 건수.
- 메타: PK 컬럼명, "전체 N개 중 M개 검사 (제외: …)". **무엇을 검사 안 했는가는 1급 정보 — 작게 묻지 말 것.**

**화면 C — 집계** (지표 한눈에)
- 5개 수치 동시 노출: 전체 / 일치 / 불일치 / As-Is만 / To-Be만.
- PK 중복 경고(있을 때만, 눈에 띄게).

**화면 D — 상세** (밀도 높은 데이터, 스캔 가능해야)
- row 목록: PK값(mono) · 상태 배지 · 다른 컬럼 미리보기.
- 필터: 문제만 / 일치만 / 전체 (기본 "문제만").
- 불일치 row 펼치면 **컬럼별 As-Is vs To-Be 값을 나란히**, **다른 부분 하이라이트**(바이트/문자). 빈 값(∅)도 명확히. 길이 차이가 시각적으로 드러나야(잔여 유닛).
- 목록은 상한(500)까지 렌더, 초과 시 안내(`.render-note`).

**화면 E — 내보내기**
- "CSV 리포트 내보내기" 액션. (출력 포맷은 고정, 버튼 스타일만)

## 6. 반드시 디자인할 상태

- 업로드: 비어있음 / 드래그 오버 / 1개만 로드 / 2개 로드 / 파싱 에러
- 설정: 검사 토글 on·off·PK, 입력 오류 메시지
- 판정: **PASS**, **FAIL**, "제외 컬럼 있는 PASS"(주의 환기)
- 집계: 정상, PK 중복 경고 있음
- 상세: 일치 row(접힘) / 불일치 row 펼침(diff 하이라이트) / 한쪽만 존재 row / 빈 값 / 렌더 상한 초과 안내 / 결과 0건
- 헤더: 언어 3종(ko/ja/en) 각각에서 레이아웃이 깨지지 않을 것 (텍스트 길이 상이)

## 7. 데이터 현실 (디자인이 견뎌야 할 조건)

- 값은 보통 긴 헥스 문자열(예 `4A6F...` 수십~수백 자). 줄바꿈/오버플로 처리 필요.
- row 수백~수천. 상세 목록은 밀도와 스캔성이 생명.
- 컬럼명은 `UNIQUE_NUMBER`, `FBWI5M` 같은 불투명한 코드형 — 길이 들쭉날쭉.
- 다국어(구현 완료): UI는 **한국어/일본어/영어** 3종 + 헤더 언어 선택. 같은 화면이 언어마다 텍스트 길이가 달라지므로(특히 영어가 길어짐) **고정폭 가정 금지** — 라벨/버튼/메트릭 카드가 줄바꿈·말줄임에도 견뎌야 함. 단어 잘림 주의. (CSV 리포트는 언어 무관 영어 고정)
